## 三层费用模型

| 费用层 | 何时收 | 用户感知 | 优化影响 |
|--------|--------|:--:|------|
| 申购/买入费 | 每次买入 | ✅直接看到 | 蚂蚁打折≈0，小额定投忽略 |
| 运作费(管理+托管+销售) | 每日从净值扣 | ❌无感知 | **长期最大成本来源** |
| 赎回/卖出费 | 卖出时 | ✅直接看到 | 持有>2年基本全免 |

> ⚠️ 用户偏好：说"运作费率"而不是笼统的"费率"，区分清楚

## 10年持有成本计算公式

### 一次性持仓（无定投）
```python
total_cost = 0
for t in range(1, years + 1):
    yr_value = holdings * (1 + r) ** (t - 1)
    total_cost += yr_value * op_rate  # op_rate = (管理+托管+销售)/100
end_value = holdings * (1 + r) ** years
```

### 每日定投
```python
# 年均余额 ≈ 累计投入 × (1+r)^(年数/2)
for yr in range(1, years + 1):
    yr_contrib = daily_amt * 365
    cumulative += yr_contrib
    avg_bal = cumulative * (1 + r) ** ((yr - 0.5) / years)
    total_cost += avg_bal * op_rate
```

## 优化决策树

### 高费率基金处理逻辑
```
费率 > 1.0%?
├── 有定投? → 🛑 停定投
│   └── 锁定期? → 期满清仓转低费指数
├── 无定投+小仓位(<3%总仓)? → ⚫ 持仓不动
├── 亏损>15%+高费率+无定投? → 🛑 等反弹（不追投）
└── 同指数有低费替代? → 卖出转低费

用户偏好: 停定投但不清仓，不折腾小仓位
```

### 优化优先序
1. **停定投** — 最高费率+有定投的基金（立即省钱）
2. **合并** — 同指数重复持有且费率差>3倍
3. **观望** — 小仓位亏损+高费率（不操作）
4. **清仓** — 锁定期满+高费率+无定投
