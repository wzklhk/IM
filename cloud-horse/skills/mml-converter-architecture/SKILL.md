# MML Converter Toolset Architecture

This describes the modular architecture of the `mml-manager` project — a web-based MML configuration management system with CLI tools.

## What is MML?

MML (Man-Machine Language) is a configuration command format used in telecom/network equipment:

```
SET TABLE_NAME:KEY1=value1,KEY2="value with spaces",KEY3=123;
ADD TABLE_NAME:KEY1=value1;
```

## Data Models (utils/table.py)

- `MmlConfig`: Represents a single MML command line. Fields: `table_name`, `cmd_type` (SET/ADD), `params` (dict), `raw` (original line).
- `MmlDataSet`: Collection of MmlConfig grouped by table_name. Methods: `add`, `get_tables`, `get_table`, etc.

## Key Design Principles

1. **One source of truth for formatting**: `utils/mml.py` has the single `quote_mml_value()` implementation — never duplicate it.
2. **Web + CLI in one package**: The Flask app (`app.py`) serves both the REST API and the SPA frontend. The `converters/` directory holds CLI-accessible format converters that reuse the same utils.
3. **Three-layer web architecture**: `controller/` (routes) → `service/` (business logic) → `dao/` (SQLite CRUD).
4. **Externalized config**: `config.yaml` is loaded by `config.py` with search order: `./config.yaml` → `~/.mml-manager/config.yaml` → `$MML_CONFIG_PATH`. All relative paths resolve based on config file location.
5. **SPA static serving**: In production, `npm run build` outputs to `converter/static/`. Flask's `app.py` serves both `/api/*` and the frontend (`/` and SPA fallback) from `static/`. No separate Nginx needed.

## Deployment Patterns

### Local (bash script)
```bash
./run.sh              # npm install → npm run build → python app.py
./run.sh build        # Only build frontend
./run.sh start        # Only start backend (requires prior build)
./run.sh dev          # Dev mode (backend + Vue dev server with hot reload)
```

### Docker (single container)
```bash
cd mml-manager
docker compose up -d
```
Port 80 → Gunicorn (4 workers) → Flask. Database persisted via `mml_data` volume at `/app/data/mml_config.db`.

### Dockerfile structure (multi-stage)
- **Stage 1** (Node 22 Alpine): Build Vue frontend → output to `converter/static/`
- **Stage 2** (Python 3.11 Slim): Copy Flask code + built static files → Gunicorn on port 80
- Full mirrors for China VPS: npm (npmmirror), pip (tuna), apt (tuna)

## Command Line Conventions

All converters follow:
```bash
python <script> input [--output path] [--table NAME] [--cmd SET|ADD]
```
- `--table/--t`: Filter/specify table name
- `--cmd/--c`: MML command type (default: SET)
- `--output/-o`: Output path (inferred from input if omitted)
- `--overwrite`: Force overwrite existing output
