# 技能开发工作流

> 从问题发现到技能发布的完整流程

## 流程图

```
发现重复工作/找到最佳实践
    ↓
创建知识卡片 (01-Obsidian-Notes/)
    ↓
编写技能文档 (02-Skill-Docs/)
    ↓
使用 skill_manage 注册为正式技能
    ↓
更新 Skills Catalog
    ↓
在实践中使用并迭代
```

## 质量检查清单

创建或更新技能前，确认：

- [ ] 步骤编号清晰，命令可以直接复制执行
- [ ] 包含验证步骤 (如何确认执行成功)
- [ ] 包含常见错误和解决方案
- [ ] 平台/环境要求注明
- [ ] 没有泄露密钥或凭据
- [ ] 与其他技能不冲突
