# Prompt 优化指南 — 面向 DeepSeek API

> 本指南基于 DeepSeek Chat API (deepseek-chat) 编写，适用于其他 OpenAI 兼容 API。

## 一、Prompt 结构黄金法则

### 1. 系统 Prompt (System Message)
**位置：** 第一条消息，设定全局行为。
**原则：** 清晰、具体、负面约束少用（优先正面引导）。

```
好：你是 DeepSeek 编程助手。回答前附上代码解释。优先使用 Python 3.11+ 语法。
差：不要废话。不要用 Markdown。不要写 Java。不要...
```

### 2. 用户 Prompt (User Message)
**结构化模板：**

```
## 目标
[一句话说清要什么]

## 上下文
[背景信息、已有代码、环境限制]

## 要求
- [格式要求]
- [质量要求]
- [约束条件]

## 示例（可选）
输入: ...
输出: ...
```

### 3. 角色锚定法
在用户消息开头显式指定角色，提升输出一致性：

```
你是有10年经验的 Kubernetes 运维专家。请解释以下问题：
```

## 二、DeepSeek API 特定优化

### 温度参数
| 任务类型 | Temperature | top_p | 说明 |
|----------|------------|-------|------|
| 代码生成 | 0.0 - 0.2 | 0.9 | 确定性优先 |
| 翻译/摘要 | 0.3 | 0.9 | 适中的创造力 |
| 创意写作 | 0.7 - 0.9 | 0.95 | 激发多样性 |
| 推理/数学 | 0.0 - 0.1 | 0.8 | 严格准确 |

### max_tokens
- 不设则 API 用默认值（通常 2048）
- 复杂任务建议：`max_tokens=4096`
- 代码生成：`max_tokens=8192`

## 三、中文 Prompt 优化技巧

### 1. 中英文混合指令
中文描述任务目标，英文/代码用于技术术语：

```
请 review 以下 Python 代码，检查 performance bottleneck、memory leak 和 type safety。
```

### 2. 少样本示例 (Few-Shot)
每个示例包含清晰的输入/输出边界：

```
将以下中文句子改写为简洁的技术文档风格。

示例：
输入：这个功能是把用户上传的图片进行压缩处理然后存到服务器上
输出：对用户上传的图片执行压缩后存储至服务器

输入：我们需要监控每台服务器的 CPU 使用率和内存占用情况
输出：
```

### 3. Chain-of-Thought 引导
```
请逐步分析以下问题。
第一步，列出已知条件。
第二步，推理可能方案。
第三步，选择最优方案并说明理由。
...
```

## 四、常见陷阱

### 陷阱 1：指令冲突
❌ "简单解释量子计算，但要覆盖所有数学推导"
→ 拆分为两个请求

### 陷阱 2：模糊的范围描述
❌ "优化这段代码"
✅ "将这段代码的时间复杂度从 O(n²) 降到 O(n)，保持相同接口"

### 陷阱 3：缺少输出格式约束
❌ 没有格式要求 → 模型自由发挥
✅ "用 JSON 格式返回：{name: string, result: number, explanation: string}"

## 五、评测方法论

简单自评标准：

| 维度 | 好 | 差 |
|------|-----|-----|
| 遵循指令 | 完全符合要求 | 遗漏关键约束 |
| 格式准确 | 输出格式一致 | 格式混乱 |
| 内容质量 | 有深度、准确 | 表面、错误 |
| 效率 | 一次到位 | 需多轮修正 |
