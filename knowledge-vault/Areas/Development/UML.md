# UML — Unified Modeling Language（统一建模语言）

## 是什么

UML 是软件工程中用于**可视化建模**的标准图形化语言，用来描述系统的结构、行为和架构。不依赖具体编程语言。

## 核心分类

### 结构图（描述系统静态结构）

| 图类型 | 说明 | 用途 |
|--------|------|------|
| **类图** | 类、接口、属性、方法、关系 | 最常用，OO设计的核心 |
| 对象图 | 某个时刻的对象快照 | 调试、理解运行时结构 |
| 组件图 | 软件组件及其依赖 | 系统架构、模块拆分 |
| 部署图 | 物理节点上的部署 | 分布式系统、硬件拓扑 |
| 包图 | 包的依赖关系 | 大项目模块组织 |
| 组合结构图 | 类内部结构 | 复杂类的内部协作 |

### 行为图（描述系统动态行为）

| 图类型 | 说明 | 用途 |
|--------|------|------|
| **用例图** | 角色、功能、系统边界 | 需求分析，最常用 |
| **时序图** | 对象间消息的时间顺序 | 协议、交互流程设计 |
| **活动图** | 业务流程、工作流 | 业务逻辑、算法流程 |
| **状态机图** | 对象状态变化 | 协议状态、订单状态 |
| 通信图 | 对象间的消息连接 | 替代时序图，强调结构 |
| 交互概览图 | 交互图的流程控制 | 组合多个交互场景 |
| 时序图 | 生命线的时间变化 | 实时系统 |

## 类图中的关系

| 关系 | 表示 | 含义 |
|------|------|------|
| 继承/泛化 | `—▷` 空心三角 | is-a |
| 实现 | `- - -▷` 虚线三角 | 接口实现 |
| 关联 | `——` 实线 | has-a，长期引用 |
| 聚合 | `—◇` 空心菱形 | 弱整体-部分（可独立存在） |
| 组合 | `—◆` 实心菱形 | 强整体-部分（同生命周期） |
| 依赖 | `- - ->` 虚线箭头 | 临时使用（参数/返回值） |

## 常用场景

1. **需求阶段** → 用例图
2. **设计阶段** → 类图、时序图、活动图
3. **架构阶段** → 组件图、部署图
4. **详细设计** → 状态机图、通信图
